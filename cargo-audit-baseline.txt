# ── RSA Marvin attack (transitive via sqlx) ──────────────────────────
#
# RUSTSEC-2023-0071 — rsa 0.9.10: Marvin Attack timing sidechannel on
# PKCS#1 v1.5 RSA decryption.
#
# Why we have it: `sqlx 0.x` is structurally designed so the meta-crate
# pulls all four driver sub-crates (`sqlx-mysql`, `sqlx-postgres`,
# `sqlx-sqlite`, `sqlx-mssql`) regardless of feature flags. Features
# gate which driver gets compiled, not which gets resolved into the
# lockfile. So sqlx-mysql is in `Cargo.lock` even though we never enable
# the mysql feature; sqlx-mysql pulls rsa for MySQL TLS/auth code.
# `cargo tree -i rsa@0.9.10 --target all` returns "nothing to print"
# — rsa is in the lockfile but in no active build-graph edge for our
# target features.
#
# Optional secondary path: arti-client 0.27 → tor-llcrypto → rsa, but
# arti-client is behind the `federation` feature (off by default).
#
# Blast radius: zero in the default build (rsa code is never compiled).
# Federation builds would link rsa but only use it inside the Tor
# handshake stack, where the Marvin sidechannel is mitigated by Tor's
# own timing defenses.
#
# Unblock: either (a) sqlx drops the always-resolves-all-drivers
# pattern, or (b) a patched rsa release lands. Track sqlx upstream.
RUSTSEC-2023-0071

# ── Soundness UB in glib (Linux only, transitive Tauri) ──────────────
#
# RUSTSEC-2024-0429 — glib <0.20.0: unsoundness in `Iterator` /
# `DoubleEndedIterator` impls for `VariantStrIter::impl_get`.
# `&p` (immutable) was passed where the C variadic out-argument expected
# `&mut p`; recent rustc optimisations discard those writes, so the
# subsequent `CStr::from_ptr(p)` dereferences NULL → crash / UB.
#
# Patched in glib >= 0.20.0. Olympus is on glib 0.18.5, transitive via:
#   tauri 2.11.2 → muda/tao/webkit2gtk/wry → gtk 0.18.2 → glib 0.18.5
# tauri 2.11.x is pinned to the gtk-rs 0.18 family; we cannot bump glib
# in isolation without ripping out the whole gtk binding stack.
#
# Runtime exposure: reachable only through `GVariant` string-iterator
# code paths inside the webview chrome (drag-drop payloads, file-chooser
# dialogs, system menu items). The first-party Axum/sqlx/ZK paths are
# unaffected. Linux-only — Windows and macOS Tauri builds don't load
# this glib code.
#
# This is the single advisory that is more than "unmaintained" — it is
# exploitable soundness UB. Listed separately so a future reviewer
# doesn't lump it back into the gtk-rs cluster.
#
# Unblock: same trigger as the gtk-rs cluster — tauri release that
# pulls gtk-rs >= 0.19/0.20.
RUSTSEC-2024-0429
