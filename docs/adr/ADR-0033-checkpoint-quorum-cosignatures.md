# ADR-0033: Checkpoint-quorum co-signatures (`OLY:CHECKPOINT:QUORUM:V1`)

- **Status:** **Accepted — 2026-06-16.** Phase 1 (helper + verifier + golden
  vector + independent JS verifier, **no producer**) is implemented in
  `src-tauri/src/quorum/checkpoint.rs`. Phase 2 (the producer + `chain_id`/epoch
  binding) is specified in §5 but not yet built.
- **Builds on:**
  - ADR-0032 (retire witness-over-root) — chose to provide institutional root
    co-signing by reusing the SBT-quorum primitive under a new domain tag rather
    than a second Ed25519 stack. This ADR is that scheme.
  - ADR-0009 (Poseidon / BJJ-EdDSA) — signatures are BabyJubJub EdDSA-Poseidon
    over BN254, the curve that is provable in-circuit (the gated
    `federation_quorum` circuit).
  - ADR-0005 (length-prefixed framing) — `lp(x)` is the normative encoding for
    every variable-length field in the co-sign message.
- **Does not change:** any leaf/node hash, any circuit, any vkey, any ceremony
  manifest, the SBT quorum message (`OLY:SBT:QUORUM:V2`), or the existing
  **single-signer** checkpoint signature (`federation::verify::verify_checkpoint_signature`,
  where one authority signs the bare `ledger_root`). No migration-class hash event.

## Context

Olympus emits a signed checkpoint that binds the Poseidon `snapshot_root` under
the node's own BJJ authority key (single signer), anchors a digest externally
(RFC 3161 / Rekor / OTS), and — under the `federation` feature — gossips a
`PeerCheckpoint` to peers (ADR-0031). What is missing is a way for **M of N
pinned institutions to jointly attest a checkpoint root** — the property the
deleted witness-over-root scaffold aimed at (ADR-0032).

The live SBT-quorum primitive (`quorum::verify_quorum`) already solves M-of-N
co-signing correctly: pinned signer set, member-only counting, dedup by
normalized key, threshold bound into the signed message with a `≥ 1` floor,
fail-closed, offline-verifiable, full adversarial test suite. Its message is a
generic length-prefixed BLAKE3 fold; it binds a `commit_id` today. Checkpoint-
quorum is the same primitive over a **ledger root** under a disjoint domain tag.

## Decision

### Domain tag

`CHECKPOINT_QUORUM_PREFIX = b"OLY:CHECKPOINT:QUORUM:V1"`, disjoint from
`OLY:SBT:QUORUM:V2` (SBT quorum), the bare-`commit_id` single-issuer signature,
and `OLY:SBT:REVOKE:V1` (revocation). The tag enters the BLAKE3 pre-image, so a
signature minted in one role can never verify in another.

### Co-sign message (normative)

Every signer signs the BN254 scalar `msg`:

```text
msg = Fr_le( BLAKE3(
    "OLY:CHECKPOINT:QUORUM:V1"
  | u32_be(len(root_dec)) || root_dec
  | u32_be(threshold)
  | u32_be(N) || for each canonical signer in sorted order:
                   u32_be(len(x)) || x || u32_be(len(y)) || y
) )
```

- `root_dec = fr_to_decimal(root)` — the canonical decimal of the checkpoint
  root field element. Canonicalization makes `"007"`, `"7"`, and any value `≥ r`
  collapse to one representation, so they cannot produce distinct messages.
- `threshold` is a **`u32`** at the API boundary (not `usize`), so the value
  bound into the fixed 4-byte field can never silently truncate. `N` is the
  count of distinct canonical signers.
- Signers are normalized (`parse_fr → fr_to_decimal`), **deduplicated**, and
  **sorted** as `(x, y)` decimal-string tuples (a `BTreeSet<(String, String)>`),
  so the digest is independent of input ordering and of non-canonical encodings.
- `Fr_le` reduces the 32-byte BLAKE3 digest as a little-endian integer modulo
  the BN254 scalar field `r` (`Fr::from_le_bytes_mod_order`). The signature
  scheme is BabyJubJub EdDSA-Poseidon over that `msg`.

Binding `root`, `threshold`, and the signer set into the message (the R3-01
property the SBT quorum already has) means none of the three can be altered after
the fact without invalidating every collected signature — the verifier then
counts zero and reports `satisfied = false` (fail-closed).

### Verification (`verify_checkpoint_quorum`)

A co-signature counts toward the quorum iff ALL hold:

1. its signer is a member of the pinned `signers` set;
2. the BJJ-EdDSA signature verifies over `msg` (which enforces subgroup +
   malleability guards in `baby_jubjub::verify_signature`);
3. the signer has not already been counted (distinctness, keyed on the
   normalized pubkey — one key under two labels counts once).

`satisfied` requires `valid_signatures ≥ threshold` **and** `threshold ≥ 1`. Any
parse failure on a signer or signature drops that entry rather than aborting.

### Conformance

The byte layout is pinned by a golden vector
(`verifiers/test_vectors/checkpoint_quorum_vectors.json`, a per-concern fixture
file alongside `parser_vectors.json` / `redaction_vectors.json`) generated by
`cargo run -p olympus-desktop --example gen_checkpoint_quorum_vectors` and
verified by two independent implementations: the Rust authoritative verifier
(`golden_vectors_match_committed_file`) and an independent Node verifier
(`verifiers/javascript/test_checkpoint_quorum.js`) that re-derives `msg` from
scratch with circomlibjs. Cases cover satisfied/insufficient quorum, non-member
rejection, duplicate dedup, wrong-root, tampered signature, threshold downgrade,
and the zero-threshold invariant.

## Consequences

- Institutional root co-signing reuses one hardened co-sign surface (the
  `quorum` module) instead of a parallel scheme, and stays on the BN254-provable
  BJJ curve, so an in-circuit "≥ M institutions attested this root" proof remains
  reachable.
- The scheme is disjoint from, and does not replace, the single-signer checkpoint
  signature; a node can carry both.

## Phase 2 (not in this ADR's implementation)

- **`chain_id` / epoch binding.** Phase 1 binds the root; Phase 2 binds a
  `chain_id` (genesis/ledger identity) and an epoch/height so a co-signature for
  one ledger or epoch cannot be replayed onto another. This belongs with the
  producer, which has the real checkpoint's `tree_size`/identity to bind. A
  payload change at that point bumps the tag to `OLY:CHECKPOINT:QUORUM:V2`.
- **Producer.** Co-sign real `own_checkpoints` roots and collect peer
  co-signatures over Tor (the checkpoint analogue of `federation::cosign`), with
  the signer set sourced from the trusted-peer registry and the threshold from an
  env default (clamped `≥ 1`) with a per-checkpoint override — mirroring
  `OLYMPUS_FEDERATION_QUORUM_THRESHOLD`. Persist the collected signatures with the
  pinned signer set + threshold for reproducible offline verification, via a new
  `migrations/` table.
