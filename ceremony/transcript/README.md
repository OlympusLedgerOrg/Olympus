# Ceremony Transcript

## Dev Proving Key

`dev_redaction_validity_final.zkey` is a **development-only** Groth16 proving
key generated by a single-participant ceremony (`dev-alpha`). It exists solely
to allow local development and CI without re-running circuit setup.

**This key MUST NOT be used in production.** A production deployment requires
a multi-party trusted setup ceremony with independent participants. See
`ceremony/README.md` and `docs/adr/0003-unified-proof-system.md` for the
planned production ceremony process.

The dev ceremony transcript and contribution hashes are recorded in
`dev-transcript.json` for auditability.
