# Third-Party Licenses

Olympus is distributed under the terms of the project license (see [LICENSE](LICENSE)).
This document summarises the licenses of bundled third-party components and points
at the machine-readable Software Bill of Materials (SBOM) that CI publishes on every
commit.

The **authoritative** list of dependencies and licenses is the CycloneDX SBOM
generated by the `supply-chain` job in [.github/workflows/ci.yml](.github/workflows/ci.yml).
Pull the latest `sbom-*` workflow artifact for the full enumeration; this document
exists for human review of the *categories* of obligation we are subject to.

## Note on iden3 / GPL-3.0 circuit templates

The Circom source files under [proofs/circuits/](proofs/circuits/) `#include`
`circomlib` templates (e.g. `circomlib/circuits/poseidon.circom`). The
`.circom` source files are therefore derivative works of `circomlib` and
inherit its GPL-3.0 license. We treat them as such.

What ships in the Olympus desktop binary is **not** Circom or snarkjs:

- The runtime prover/verifier is `ark-circom` + `ark-groth16` (Apache-2.0 / MIT).
- The artifacts consumed at runtime — `.r1cs`, `.wasm` witness generator, the
  exported `.ark.zkey`, and `*_vkey.json` — are produced by the build-time
  toolchain and consumed by the Rust runtime; the GPL toolchain itself is
  not linked, loaded, or executed by the released binary.

This is the same posture taken by other production deployments built on
`circomlib` (Aztec, Polygon ID, etc.) and matches iden3's published intent
for the library. No court has tested this framing.

## Tracked GPL-3.0 exception (`poseidon-rs`)

Olympus currently links one GPL-3.0 crate transitively: `poseidon-rs@0.0.8`,
pulled in by `babyjubjub-rs@0.0.11`. Olympus's Poseidon hot path uses the
vendored permissive [crates/light-poseidon/](crates/light-poseidon/)
(Apache-2.0); `poseidon-rs` is linked because `babyjubjub-rs` depends on it,
not because Olympus code calls it.

A narrow exception is declared in [deny.toml](deny.toml) so CI passes today.
GPL-3.0 remains banned globally — only this exact crate/version is allowed.

**Replacement plan**: vendor a permissive BJJ-EdDSA implementation at
`crates/babyjubjub-permissive/` built on `taceo-ark-babyjubjub`
(MIT/Apache-2.0, BJJ curve) + the existing `light-poseidon` (Apache-2.0)
+ a ~200 LOC EdDSA construction matching circomlib's wire format, with a
JS-side parity test against `circomlibjs` to guard against encoding drift.
Once that lands, `babyjubjub-rs` (and `poseidon-rs` with it) is dropped
from `Cargo.toml` and the exception is removed from `deny.toml`.

After that change, the shipped Olympus desktop binary contains **no
GPL-3.0 code**. Build-time GPL tools (`circom`, `snarkjs`, `circomlib`,
`circomlibjs`) remain GPL-3.0 but are not distributed to end users.

## What to bundle in the installer

Tauri's bundler emits license files when listed under `tauri.bundle.resources`
(or platform-specific `bundle.windows.wix.licenseFile` /
`bundle.macOS.licenseFile`). The `LICENSES/` directory should contain at
minimum:

- `LICENSE-APACHE-2.0`
- `LICENSE-MIT`
- `LICENSE-BSD-3-CLAUSE`
- `LICENSE-POSTGRESQL`
- `LICENSE-LGPL-2.1` (with a pointer to `crates/glib-0.18.5-patched/LICENSE`)
- `LICENSE-GPL-3.0` (for `proofs/circuits/*.circom`)

## Verifying compliance locally

```bash
# Rust dependency licenses (requires cargo-deny; see deny.toml)
cargo deny check licenses

# Frontend dependency licenses
pnpm --filter app/public-ui exec license-checker --summary

# Authoritative SBOM (matches CI)
cargo cyclonedx --format json --all
```
